NRF通信协议

数据头:
deviceId		1	
code			2	
msgType			1	
	0:注册设备  绑定deviceId 和 Addr	设备主动发送到中转站,中转站收到则绑定
	1:命令 中转站 -> 设备				中转站转发给设备,15ms内中转站等待设备回复
	2:回复 设备 -> 中转站				
	3:推送 设备 -> 中转站				设备主动推送数据给中转站,15ms内等待中转站应答,超时则继续推送
	4:心跳 设备 -> 中转站
	//增强型不用回 4.回复 中转站 -> 设备				中转站回复设备的推送
	
两种类型数据体:
第一种注册设备,4个字节传送地址
第二种传输字符串(应用层可用base64转换),最大支持28字节字符串


中断(关) -> 接收解析装载数据 -> TX循环发送数据


中转站:
0 -> 不回	-> 执行设备绑定 -> 发送mqtt
1 x
2 -> 不回	-> 携带pcode发送mqtt
3 -> 不回	-> 发送mqtt
